# Data Governance

## Raw Datasets — Not Committed to This Repository

Raw partner datasets (Maskott/Tactileo, Inokufu/Becomino) are **never committed** to this public repository.

Only synthetic samples are included under `data/samples/`. These samples are generated by `affectlog generate-synthetic` and contain no real personal data.

## How to Use Real Datasets Locally

1. Obtain a signed data-sharing agreement with the data controller (Maskott / Inokufu).
2. Place files under `data/raw/maskott/` or `data/raw/becomino/`.
3. These directories are git-ignored — they will never be committed.
4. Run `affectlog validate-csv --input data/raw/maskott/your_file.csv --schema maskott_csv_v1`.

## Residual Re-identification Risk

Even after pseudonymisation:

- Small institutional cohorts (EntityUaiCode) may be re-identifiable through attribute combination.
- Temporal patterns (AccessDate) may correlate with real-world events.
- Sparsity of interactions may allow inference in very small datasets.

**Mitigation:** Apply k-anonymity checks before any aggregate sharing. Review the `privacy_report.json` in each run.

## Audit Outputs

All default outputs contain:
- Hashed/pseudonymised identifiers only
- Aggregate metrics (Gini, Coverage@K, sparsity)
- No individual-level raw data
- Config hash for reproducibility

## GDPR Lawful Basis

Processing of Maskott/Tactileo data is conducted under:
- Legitimate interest (Article 6(1)(f)) for research and AI assessment
- Data minimisation principle (Article 5(1)(c))
- Pseudonymisation measure (Article 32)

This tool does not grant a lawful basis — the data controller is responsible for establishing and documenting the basis.
